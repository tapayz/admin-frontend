주요 기능:

  • 실시간 암호화폐 결제 처리
  • 자동 환율 계산 및 적용
  • 고객 관리 시스템
  • 인보이스 발행 및 추적
  • 다중 네트워크 지원

주요 지원 자산

  • TRX: TRON 네이티브 코인

  • ETH: Ethereum 네이티브 코인
  • BTC: Bitcoin
  • 기타: 다양한 ERC-20, TRC-20 토큰

2.4 콜백 시그니처 인증

콜백 요청의 무결성과 신뢰성을 보장하기 위해 HMAC-SHA256 기반의 시그니처 인증을 사용합니다.

signature = HMAC-SHA256(api_key, "POST|timestamp|JSON_payload")

시그니처 검증 예시 (Node.js):

const crypto = require('crypto');

function verifyCallbackSignature(method, timestamp, payload, signature, apiKey) {
  const raw = `${method.toUpperCase()}|${timestamp}|${JSON.stringify(payload)}`;
  const expectedSignature = crypto
    .createHmac('sha256', apiKey)
    .update(raw)
    .digest('hex');

  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSignature)
  );
}

3. 고객 관리 API

3.1 고객 등록

새로운 고객을 시스템에 등록합니다. 서비스를 이용하기 위해서는 고객 등록이 필수입니다.

{
  "name": "홍길동",
  "country": "KR",
  "email": {
    "value": "hong@example.com",
    "desc": "업무용 이메일"
  },
  "phone": {
    "value": "010-1234-5678",
    "desc": "개인 휴대폰"
  },
  "memo": "VIP 고객"
}

3.2 고객 정보 조회

  • 이름으로 조회: GET /customer/{name}
  • 상세 정보 조회: GET /customer/detail?customerId=123

3.3 고객 목록 조회

페이지네이션을 지원하는 고객 목록 조회

파라미터 설명
page 페이지 번호 (기본값: 1)
limit 페이지당 항목 수 (기본값: 20, 최대: 100)
search 검색 키워드 (이름, 이메일, 전화번호)
sortBy 정렬 기준 (name, createdAt, updatedAt)
sortOrder 정렬 순서 (asc, desc)

3.4 고객 정보 수정

  • 이름 수정: POST /customer/name
  • 연락처 추가: POST /customer/contact
  • 메모 추가: POST /customer/memo

3.5 고객 아이콘 업로드

고객의 프로필 아이콘을 업로드합니다.

Endpoint: POST /customer/icon
Content-Type: multipart/form-data

지원 형식:

  • JPEG, PNG, GIF
  • 최대 파일 크기: 5MB
  • 권장 해상도: 512x512px

4. 인보이스 발행 및 관리

4.1 인보이스 발행

암호화폐 결제를 위한 인보이스를 발행합니다.

{
  "customerId": 123,
  "customerName": "홍길동",
  "title": "USDT 10만원 구매",
  "priceAtRequest": 1350.5,
  "fiatAssetId": 1,
  "cryptoAssetId": 1001,
  "cashAmount": 100000,
  "expiredSecond": 600
}

필드 설명:

  • customerId: 고객 ID
  • customerName: 고객 이름
  • title: 인보이스 제목
  • priceAtRequest: 요청 시점의 환율
  • fiatAssetId: 법정화폐 자산 ID (1: KRW)
  • cryptoAssetId: 암호화폐 자산 ID (1001: USDT-TRC20)
  • cashAmount: 법정화폐 금액
  • expiredSecond: 만료 시간(초, 기본값: 600)

응답 예시:

{
  "success": true,
  "data": {
    "invoiceId": "inv_1234567890abcdef",
    "paymentUrl": "https://pay.your-domain.com/invoice/inv_1234567890abcdef",
    "depositAddress": "TQn9Y2khEsLJW1ChVWFMSMeRDow5KcbLSE",
    "network": "TRX",
    "cryptoAmount": "74.074074",
    "cashAmount": 100000,
    "exchangeRate": 1350.5,
    "status": "Ready",
    "expiresAt": "2024-01-15T10:40:00Z"
  }
}

7. 응답 코드 및 에러 처리

7.1 HTTP 상태 코드

API 요청에 대한 HTTP 상태 코드입니다.

  • 성공 응답: 200 OK, 201 Created
  • 클라이언트 오류: 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 409 Conflict, 429 Too Many Requests
  • 서버 오류: 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable

7.2 에러 코드 정의

시스템에서 정의된 상세 에러 코드입니다.

  • 인증 관련: AUTH_REQUIRED, INVALID_API_KEY, API_KEY_EXPIRED, INSUFFICIENT_PERMISSION
  • 고객 관리: CUSTOMER_NOT_FOUND, CUSTOMER_NAME_EXISTS, INVALID_CUSTOMER_DATA
  • 인보이스 관련: INVOICE_NOT_FOUND, INVOICE_EXPIRED, PRICE_FLUCTUATION_WARNING, CANNOT_CANCEL
  • 거래 관련: TRANSACTION_NOT_FOUND, INVALID_ADDRESS, INSUFFICIENT_BALANCE, MINIMUM_AMOUNT_ERROR, WITHDRAW_DISABLED

7.3 에러 응답 형식

오류 발생 시 표준화된 응답 형식입니다.

{
  "success": false,
  "error": {
    "code": "CUSTOMER_NOT_FOUND",
    "message": "고객을 찾을 수 없습니다.",
    "details": "customerId: 999는 존재하지 않는 고객입니다."
  },
  "timestamp": "2024-01-15T12:00:00Z",
  "requestId": "req_1234567890"
}

7.4 일반적인 에러 해결 방법

  • 인증 오류: API 키 및 헤더 형식 확인, 만료 여부, IP 화이트리스트 설정 확인
  • 요청 형식 오류: Content-Type 헤더, JSON 형식, 필수 필드 및 데이터 타입 확인

8. 웹훅 (Webhook)

웹훅을 통해 실시간으로 이벤트 알림을 받을 수 있습니다. 인보이스 및 거래 상태 변화를 추적하여 비즈니스 로직을 처리할 수 있습니다.

8.1 웹훅 설정

파트너 대시보드에서 웹훅 URL 등록 (https://your-domain.com/webhook 형식)

8.2 인보이스 Webhook

인보이스 상태가 변경될 때마다 /callback/update-invoice 엔드포인트로 실시간 알림을 전송합니다. 결제 완료, 만료, 실패 등의 상태 변경을 즉시 감지하여 비즈니스 로직을 처리할 수 있습니다.

10. 운영 가이드

10.1 모니터링 및 로깅

API 응답 시간, 에러율, 거래 성공률 등을 모니터링하고, 모든 요청/응답을 로깅하는 것이 중요합니다.

10.2 레이트 리밋

API 호출에는 기본적으로 분당 60회 등의 제한이 있습니다. 초과 시 HTTP 429 응답을 받으며, 지수 백오프 방식의 재시도를 권장합니다.

10.3 장애 대응

API 응답 지연, 웹훅 전송 실패, 네트워크 장애 등 일반적인 장애 상황에 대한 대응 방안이 설명되어 있습니다.

11. FAQ 및 문제해결

자주 묻는 질문과 함께, 401 Unauthorized 에러, 환율 오류, 웹훅 수신 실패 등 일반적인 문제에 대한 해결 방법을 안내합니다.

Q. 401 Unauthorized 에러가 발생합니다.

A. API 키, IP 화이트리스트 설정, API 접근 권한 등을 다시 확인해 주세요. 운영 플랫폼에서 API 키 상태를 점검할 수 있습니다.

Q. 웹훅을 받지 못하고 있습니다.

A. 아래 사항을 점검하세요.

  • 등록된 콜백 URL 접근 가능 여부 확인
  • HTTPS 설정 확인
  • 방화벽 설정 확인
  • 시그니처 검증 로직 확인
