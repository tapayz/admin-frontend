2. Authentication & Security

2.1 API Key Authentication

Include your issued API key in the Authorization header for every API request.

Authorization: YOUR_API_KEY

2.2 Required Request Headers

Mandatory Headers

Authorization: YOUR_API_KEY
Content-Type: application/json
Accept: application/json

2.3 Security Guidelines

API Key Security

  • Only use API keys on the server side
  • Never embed keys in client-side code
  • Store keys in environment variables or secure vaults
  • Rotate keys on a regular schedule

Network Security

  • Enforce HTTPS connections
  • Use SSL/TLS 1.2 or higher
  • Enable certificate validation

Data Security

  • Avoid logging sensitive information
  • Encrypt customer data at rest
  • Schedule regular security audits

2.4 Callback Signature Verification

We use HMAC-SHA256 signatures to guarantee the integrity and authenticity of callback requests.

signature = HMAC-SHA256(api_key, "POST|timestamp|JSON_payload")

Verification Example (Node.js)

const crypto = require('crypto');

function verifyCallbackSignature(method, timestamp, payload, signature, apiKey) {
  const raw = `${method.toUpperCase()}|${timestamp}|${JSON.stringify(payload)}`;
  const expectedSignature = crypto
    .createHmac('sha256', apiKey)
    .update(raw)
    .digest('hex');

  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSignature)
  );
}

4. Invoice Issuance & Management

4.1 Create Invoice

Issue an invoice to accept a cryptocurrency payment.

{
  "customerId": 123,
  "customerName": "Hong Gil-dong",
  "title": "Purchase 100,000 KRW in USDT",
  "priceAtRequest": 1350.5,
  "fiatAssetId": 1,
  "cryptoAssetId": 1001,
  "cashAmount": 100000,
  "expiredSecond": 600
}

Field Description

  • customerId: Customer identifier
  • customerName: Customer name
  • title: Invoice title
  • priceAtRequest: FX rate at request time
  • fiatAssetId: Fiat asset ID (1 = KRW)
  • cryptoAssetId: Crypto asset ID (1001 = USDT-TRC20)
  • cashAmount: Fiat amount
  • expiredSecond: Expiration in seconds (default: 600)

Sample Response

{
  "success": true,
  "data": {
    "invoiceId": "inv_1234567890abcdef",
    "paymentUrl": "https://pay.your-domain.com/invoice/inv_1234567890abcdef",
    "depositAddress": "TQn9Y2khEsLJW1ChVWFMSMeRDow5KcbLSE",
    "network": "TRX",
    "cryptoAmount": "74.074074",
    "cashAmount": 100000,
    "exchangeRate": 1350.5,
    "status": "Ready",
    "expiresAt": "2024-01-15T10:40:00Z"
  }
}

7. Response Codes & Error Handling

7.1 HTTP Status Codes

Standard HTTP status codes returned by the API.

  • Success: 200 OK, 201 Created
  • Client errors: 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 409 Conflict, 429 Too Many Requests
  • Server errors: 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable

7.2 Error Code Definitions

Detailed application-level error codes.

  • Authentication: AUTH_REQUIRED, INVALID_API_KEY, API_KEY_EXPIRED, INSUFFICIENT_PERMISSION
  • Customer: CUSTOMER_NOT_FOUND, CUSTOMER_NAME_EXISTS, INVALID_CUSTOMER_DATA
  • Invoice: INVOICE_NOT_FOUND, INVOICE_EXPIRED, PRICE_FLUCTUATION_WARNING, CANNOT_CANCEL
  • Transaction: TRANSACTION_NOT_FOUND, INVALID_ADDRESS, INSUFFICIENT_BALANCE, MINIMUM_AMOUNT_ERROR, WITHDRAW_DISABLED

7.3 Error Response Format

Standard response payload for failed requests.

{
  "success": false,
  "error": {
    "code": "CUSTOMER_NOT_FOUND",
    "message": "Customer not found",
    "details": "customerId: 999 does not exist."
  },
  "timestamp": "2024-01-15T12:00:00Z",
  "requestId": "req_1234567890"
}

7.4 Common Troubleshooting Tips

  • Authentication errors: verify API key, header format, expiration, and IP whitelist settings
  • Request format issues: confirm the Content-Type header, JSON structure, required fields, and data types

11. FAQ & Troubleshooting

Frequently asked questions including solutions for 401 Unauthorized errors, FX discrepancies, webhook issues, and more.

Q. I received a 401 Unauthorized response.

A. Double-check your API key, IP whitelist, and permissions. You can verify API key status in the operations platform.

Q. Webhook notifications are not arriving.

A. Verify the HTTPS configuration of the registered callback URL, inspect firewall settings, and confirm signature validation logic.
