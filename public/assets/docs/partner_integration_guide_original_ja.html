1. はじめに

1.1 サービス概要

弊社の暗号通貨決済ソリューションは、企業が簡単かつ安全に暗号通貨決済を受け入れることができるサービスです。

主な機能：

  • リアルタイム暗号通貨決済処理
  • 自動為替レート計算・適用
  • 顧客管理システム
  • インボイス発行・追跡
  • マルチネットワーク対応
  • 統計・レポート機能

1.2 対応暗号通貨・ネットワーク

対応ネットワーク

  • TRON (TRX): TRC-20トークンを含む
  • Ethereum (ETH): ERC-20トークンを含む（順次対応）
  • Bitcoin (BTC): ビットコインネットワーク（順次対応）
  • Ripple (XRP): リップルネットワーク（順次対応）

主な対応資産

  • TRX: TRONネイティブコイン
  • USDT-TRC20: Tether USD（TRONネットワーク）
  • ETH: Ethereumネイティブコイン
  • BTC: Bitcoin
  • その他: 各種ERC-20、TRC-20トークン

1.3 パートナー登録・APIキー発行

パートナー登録手続きとAPIキー発行についてのご案内です。

登録手続き：

  1. パートナー登録申請書の提出
  2. 本人確認・事業者登録証明書の審査
  3. 契約書の締結
  4. APIキー発行（運用プラットフォーム、マイページで確認可能）
  5. テスト環境アクセス提供

1.4 開発環境設定

Base URL:

2. 認証とセキュリティ

2.1 APIキー認証方式

すべてのAPIリクエストには、発行されたAPIキーをAuthorizationヘッダーに含める必要があります。

Authorization: YOUR_API_KEY

2.2 リクエストヘッダー設定

必須ヘッダー：

Authorization: YOUR_API_KEY
Content-Type: application/json
Accept: application/json

2.3 セキュリティガイドライン

APIキーセキュリティ：

  • APIキーはサーバーサイドでのみ使用
  • クライアントサイドコードへの組み込み禁止
  • 環境変数またはセキュアストレージに保存
  • 定期的なキーローテーション推奨

ネットワークセキュリティ：

  • HTTPS通信必須
  • SSL/TLS 1.2以上使用
  • 証明書検証の有効化

データセキュリティ：

  • 機密情報のログ記録禁止
  • 顧客情報の暗号化保存
  • 定期的なセキュリティ監査

2.4 コールバック署名認証

コールバックリクエストの完全性と信頼性を保証するため、HMAC-SHA256ベースの署名認証を使用します。

signature = HMAC-SHA256(api_key, "POST|timestamp|JSON_payload")

署名検証例（Node.js）：

const crypto = require('crypto');

function verifyCallbackSignature(method, timestamp, payload, signature, apiKey) {
  const raw = `${method.toUpperCase()}|${timestamp}|${JSON.stringify(payload)}`;
  const expectedSignature = crypto
    .createHmac('sha256', apiKey)
    .update(raw)
    .digest('hex');

  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSignature)
  );
}

2.5 IPホワイトリスト（オプション）

セキュリティ強化のため、特定のIPアドレスからのみAPIアクセスを許可することができます。

新しい顧客をシステムに登録します。サービスをご利用いただくには顧客登録が必須です。

{
  "name": "田中太郎",
  "country": "JP",
  "email": {
    "value": "tanaka@example.com",
    "desc": "業務用メール"
  },
  "phone": {
    "value": "090-1234-5678",
    "desc": "個人携帯"
  },
  "memo": "VIP顧客"
}

3.2 顧客情報照会

  • 名前による照会: GET /customer/{name}
  • 詳細情報照会: GET /customer/detail?customerId=123

3.3 顧客リスト照会

ページネーション対応の顧客リスト照会

パラメータ 説明
page ページ番号（デフォルト: 1）
limit ページあたり項目数（デフォルト: 20、最大: 100）
search 検索キーワード（名前、メール、電話番号）
sortBy ソート基準（name、createdAt、updatedAt）
sortOrder ソート順序（asc、desc）

3.4 顧客情報修正

  • 名前修正: POST /customer/name
  • 連絡先追加: POST /customer/contact
  • メモ追加: POST /customer/memo

3.5 顧客アイコンアップロード

顧客のプロフィールアイコンをアップロードします。

Endpoint: POST /customer/icon
Content-Type: multipart/form-data

対応形式：

  • JPEG、PNG、GIF
  • 最大ファイルサイズ: 5MB
  • 推奨解像度: 512x512px

4. インボイス発行・管理

4.1 インボイス発行

暗号通貨決済のためのインボイスを発行します。

{
  "customerId": 123,
  "customerName": "田中太郎",
  "title": "USDT 10万円購入",
  "priceAtRequest": 155.5,
  "fiatAssetId": 3,
  "cryptoAssetId": 1001,
  "cashAmount": 100000,
  "expiredSecond": 600
}

フィールド説明：

  • customerId: 顧客ID
  • customerName: 顧客名
  • title: インボイスタイトル
  • priceAtRequest: リクエスト時点の為替レート
  • fiatAssetId: 法定通貨資産ID（3: JPY）
  • cryptoAssetId: 暗号通貨資産ID（1001: USDT-TRC20）
  • cashAmount: 法定通貨金額
  • expiredSecond: 有効期限（秒、デフォルト: 600）

レスポンス例：

{
  "success": true,
  "data": {
    "invoiceId": "inv_1234567890abcdef",
    "paymentUrl": "https://pay.your-domain.com/invoice/inv_1234567890abcdef",
    "depositAddress": "TQn9Y2khEsLJW1ChVWFMSMeRDow5KcbLSE",
    "network": "TRX",
    "cryptoAmount": "643.086",
    "cashAmount": 100000,
    "exchangeRate": 155.5,
    "status": "Ready",
    "expiresAt": "2024-01-15T10:40:00Z"
  }
}

7. レスポンスコード・エラー処理

7.1 HTTPステータスコード

APIリクエストのHTTPステータスコードです。

  • 成功レスポンス： 200 OK、201 Created
  • クライアントエラー： 400 Bad Request、401 Unauthorized、403 Forbidden、404 Not Found、409 Conflict、429 Too Many Requests
  • サーバーエラー： 500 Internal Server Error、502 Bad Gateway、503 Service Unavailable

7.2 エラーコード定義

システムで定義された詳細エラーコードです。

  • 認証関連： AUTH_REQUIRED、INVALID_API_KEY、API_KEY_EXPIRED、INSUFFICIENT_PERMISSION
  • 顧客管理： CUSTOMER_NOT_FOUND、CUSTOMER_NAME_EXISTS、INVALID_CUSTOMER_DATA
  • インボイス関連： INVOICE_NOT_FOUND、INVOICE_EXPIRED、PRICE_FLUCTUATION_WARNING、CANNOT_CANCEL
  • 取引関連： TRANSACTION_NOT_FOUND、INVALID_ADDRESS、INSUFFICIENT_BALANCE、MINIMUM_AMOUNT_ERROR、WITHDRAW_DISABLED

7.3 エラーレスポンス形式

エラー発生時の標準化されたレスポンス形式です。

{
  "success": false,
  "error": {
    "code": "CUSTOMER_NOT_FOUND",
    "message": "顧客が見つかりません。",
    "details": "customerId: 999は存在しない顧客です。"
  },
  "timestamp": "2024-01-15T12:00:00Z",
  "requestId": "req_1234567890"
}

7.4 一般的なエラー解決方法

  • 認証エラー： APIキー・ヘッダー形式確認、有効期限、IPホワイトリスト設定確認
  • リクエスト形式エラー： Content-Typeヘッダー、JSON形式、必須フィールド・データタイプ確認

8. ウェブフック（Webhook）

ウェブフックを通じてリアルタイムでイベント通知を受け取ることができます。インボイス・取引ステータスの変化を追跡してビジネスロジックを処理できます。

8.1 ウェブフック設定

パートナーダッシュボードでウェブフックURL登録（https://your-domain.com/webhook形式）

8.2 インボイスWebhook

インボイスのステータスが変更されるたびに/callback/update-invoiceエンドポイントへリアルタイム通知を送信します。決済完了、期限切れ、失敗などのステータス変更を即座に検知してビジネスロジックを処理できます。

10. 運用ガイド

10.1 モニタリング・ログ記録

APIレスポンス時間、エラー率、取引成功率などをモニタリングし、すべてのリクエスト・レスポンスをログ記録することが重要です。

APIレスポンス遅延、ウェブフック送信失敗、ネットワーク障害など、一般的な障害状況への対応方案が説明されています。

11. FAQ・トラブルシューティング

よくある質問とともに、401 Unauthorizedエラー、為替レートエラー、ウェブフック受信失敗など、一般的な問題への解決方法をご案内します。

Q. 401 Unauthorizedエラーが発生します。

A. APIキー、IPホワイトリスト設定、APIアクセス権限などを再度ご確認ください。運用プラットフォームでAPIキーの状態を点検できます。

Q. ウェブフックを受信できません。

A. 下記の事項をご点検ください。

  • 登録されたコールバックURLのアクセス可否確認
  • HTTPS設定確認
  • ファイアウォール設定確認
  • 署名検証ロジック確認
